contains

  • contains<T>(array: DynamicExpression<T[]>, element: Element<T>): Operation<boolean>
    • Defined in core/src/operations/array.ts:71

    Checks if the array contains the given element.

    The comparisons are done like this:

    1. Check if the type is the same;
    2. If it's a primitive, just return a == b;
    3. If it's a map (object), compare every (key, value) pair, their order doesn't matter.
    4. If it's an array, compare all values (the order matters).

    This performs a deep comparison.

    Type parameters

    • T

    Parameters

    • array: DynamicExpression<T[]>

      the ContextNode or Operation referring to the array

    • element: Element<T>

      the element to look for

    Returns Operation<boolean>

    an instance of Operation, i.e. an operation that results in a boolean when run by the frontend.

union

  • union(...arrays: DynamicExpression<any[]>[]): Operation<any[]>
    • Defined in core/src/operations/array.ts:103

    Concatenates a series of arrays in a single array.

    Example:

    const myArrays = createContext('myArrays', {
      array1: [1, 2, 3],
      array2: [4, 5, 6],
      array3: [7, 8, 9],
    })

    const MyScreen = () => (
      <Text>
        {union(myArrays.get('array1'), myArrays.get('array2'), myArrays.get('array3'))}
      </Text>
    )

    The screen created in the code above would print "[1, 2, 3, 4, 5, 6, 7, 8, 9]".

    The arrays passed in the parameters are not changed, instead a new array is returned.

    Parameters

    • Rest ...arrays: DynamicExpression<any[]>[]

      the ContextNodes or Operations referring to the arrays.

    Returns Operation<any[]>

    an instance of Operation, i.e. an operation that results in an Array when run by the frontend.
